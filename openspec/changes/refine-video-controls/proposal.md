# 提案：优化视频播放控制体验

## 变更背景

当前 `SentencePracticeScreen` 中的视频播放控制（覆盖层播放按钮、底部控制栏）是静态显示的，会遮挡视频内容。用户希望在播放过程中自动隐藏这些控件，点击屏幕时再显示，并增加一个视频进度条以提供更好的播放反馈。

## 目标

1.  **自动隐藏控制**: 播放时，控制层（播放按钮、底部栏、顶部栏）应在无操作 3 秒后自动淡出消失。
2.  **点击交互**: 点击视频区域应能切换控制层的显示/隐藏状态。
3.  **增加进度条**: 在底部（或视频区域底部）增加视频进度条，显示当前播放进度，并随控制层一同显示/隐藏。

## 范围

- 修改 `SentencePracticeScreen` 的 UI 逻辑。
- 修改 `PracticeControls` 组件（或其父容器）以支持显隐动画。
- 添加新的 `VideoProgressIndicator` 或类似组件。

## 风险

- 动画状态管理可能会与现有的播放状态逻辑产生冲突，需要仔细处理 Timer 和状态。
- 需要确保点击事件不会与 VideoPlayer 的其他手势（如有）冲突。
