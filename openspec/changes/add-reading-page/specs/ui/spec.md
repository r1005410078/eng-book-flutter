# 规范: 阅读页面 UI

## 新增需求

### 阅读页面布局

#### 场景: 初始加载

- **当** 用户导航到阅读页面时
- **那么** 背景应该是深色的（匹配应用主题）。
- **并且** **头部** 应该显示：
  - 关闭按钮（左）。
  - 中间显示“标题”（如 "The Reunion"）和“进度”（如 "第 2 / 45 句"）。
  - “视频”图标（右），用于切换回视频模式。
- **并且** **主体** 应该以 "CHAPTER 1" 和 "TRANSCRIPT" 标签开始，紧接着是可滚动的句子列表。
- **并且** **底部** 应该显示一个悬浮的胶囊状控制栏。

### 剧本列表行为

#### 场景: 列出句子

- **给定** 一个句子列表
- **那么** 每个 **非活动** 句子行应该显示：
  - 左侧的状态图标（圆圈或对勾）。
  - 中间的句子文本（英文）和翻译（中文）。
  - 右侧的小播放按钮。
- **并且** 当前 **活动** 的句子应该显示为显眼的 **活动卡片**。

### 活动句子卡片

#### 场景: 显示活动句子

- **给定** 一个句子被选中或处于当前播放状态
- **那么** 它应该渲染在一个容器内，具有：
  - 深色/明显的背景颜色。
  - 橙色强调边框或发光效果。
  - **头部行**:
    - "Reps" 指示器（如 "2/5 REPS"，带进度条/药丸形状）。
    - 书签图标（右）。
  - **内容**:
    - 大号英文文本，带样式高亮（如特定短语的橙色下划线）。
    - 下方的中文翻译。
  - **可视化**: 一个音频可视化图标/动画。

### 底部控件

#### 场景: 控件交互

- **给定** 悬浮的底部控制栏
- **那么** 它应该包含 5 个按钮：
  1.  **上一个** 箭头（左）。
  2.  **重播/循环** 图标（旋转箭头）。
  3.  **录音/麦克风** 按钮（居中，大号，橙色背景，发光）。
  4.  **自动/循环** 图标（播放循环）。
  5.  **下一个** 箭头（右）。
