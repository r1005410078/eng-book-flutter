# 添加阅读页面 (文本模式)

## 目标

实现一个专门的“阅读页面”（文本模式）用于句子练习，让用户专注于剧本内容，练习单个句子，并在无干扰的列表视图中跟踪重复次数，匹配提供的设计图。

## 背景

目前 `SentencePracticeScreen` 侧重于视频。用户希望有一个互补的文本视图（参考 "The Reunion" 设计图），该视图：

1. 列出完整的剧本。
2. 通过展开的“活动卡片”高亮当前句子。
3. 提供特定的学习工具，如重复次数跟踪（"2/5 REPS"）和录音。
4. 允许在视频模式和阅读模式之间无缝切换。

## 架构

- **新屏幕**: `ReadingPracticeScreen`（或 `SentencePracticeScreen` 中的一种模式）。
  - _决策_: 既然它共享状态（句子数据、当前索引、播放状态），最好将其实现为一个共享数据源（Riverpod provider 或参数传递）的新路由 `ReadingPracticeScreen`。
- **路由**: 添加 `/practice/reading/:id` 路由。
- **状态管理**: 需要与视频播放器同步 `currentIndex`。通常 100LS 意味着先看视频再阅读。
- **组件**:
  - `ReadingHeader`: 自定义导航栏。
  - `TranscriptList`: 主要的可滚动区域。
  - `ActiveSentenceCard`: 聚焦句子的复杂 UI 卡片。
  - `ReadingControls`: 底部悬浮控制栏。

## 设计参考

- **主题**: 深色背景（`#1a120b` 或类似），橙色强调色（`#FF9F29`）。
- **排版**: 简洁的无衬线字体，通过字重区分强调。
- **布局**:
  - **头部**: 顶部固定或对齐。
  - **主体**: 可滚动列表。活动项自动滚动到视图中。
  - **底部**: 悬浮覆盖的控制栏。
