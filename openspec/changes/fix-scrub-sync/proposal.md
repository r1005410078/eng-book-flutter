# 提案：修复拖拽进度同步问题

## 问题描述

在 `SentencePracticeScreen` 中，手动拖拽视频进度条（Scrubbing）未能实时更新下方的字幕内容和顶部的进度条（Pills）。这可能是由于 `mockSentences` 数据覆盖范围有限，或者是同步逻辑在拖拽过程中未能正确响应位置变化。

## 原因分析 (Hypothesis)

1.  **数据范围不足**: `mockSentences` 仅定义了 0-20 秒的内容。如果将视频拖拽到 20 秒之后，循环查找逻辑无法找到匹配的句子，导致 UI 停留在最后显示的状态，或者不更新。
2.  **监听频率**: `_syncSentenceWithVideo` 依赖 `addListener`，虽然通常足够，但在 scrubbing 期间可能需要更稳健的逻辑来处理“无匹配”状态（比如在句子之间的间隙，或者超出范围）。

## 解决方案

1.  **扩展 Mock 数据**: 简单重复 Mock 数据以覆盖更长的视频时间（例如 2 分钟），确保大部分拖拽操作都能命中数据。
2.  **逻辑优化**: 在 `_syncSentenceWithVideo` 中处理“无匹配”情况。如果当前时间未命中任何句子，可以：
    - 方案 A: 保持上一个句子（当前行为，但导致看起来像没更新）。
    - 方案 B: 清空显示或显示“Waiting...”（体验不好）。
    - 方案 C: 查找**最近**的句子（Previous or Next）。
    - 方案 D (推荐): 确保 Mock 数据填满时间轴。如果是真实应用，数据应该完整。

鉴于这是 Demo 环境，**扩展 Mock 数据**是最直接有效的验证方式。
