## ADDED Requirements
### Requirement: 课程与单元学习指标

系统必须 (MUST) 为每个课程及其单元维护学习指标，至少包括练习次数、练习进度和熟练度，并在用户学习过程中持续更新。

#### Scenario: 首次进入单元产生初始指标
- **Given** 用户首次进入某课程下的某单元开始练习
- **When** 学习页完成首次有效学习行为
- **Then** 该单元练习次数应更新为 1
- **And** 该课程练习次数应同步更新
- **And** 单元与课程应生成可展示的进度与熟练度值

#### Scenario: 指标更新具有一致性
- **Given** 用户在同一学习会话中连续切句、切单元或切课程
- **When** 系统触发多次指标写入
- **Then** 指标写入必须保证最终结果反映最新学习状态
- **And** 不得因旧写入覆盖导致进度倒退或练习次数异常增长

### Requirement: 选择面板指标展示

系统必须 (MUST) 在课程/单元选择面板中展示课程与单元学习指标，帮助用户判断继续学习与复习优先级。

#### Scenario: 课程列表展示课程指标
- **Given** 用户打开课程/单元选择面板
- **When** 面板加载课程列表
- **Then** 每个课程项应显示课程练习次数、课程进度和课程熟练度
- **And** 当前课程应有明确的视觉高亮

#### Scenario: 单元列表展示状态与熟练度
- **Given** 用户已选中某课程
- **When** 面板渲染该课程的单元列表
- **Then** 每个单元应显示状态（未开始/学习中/已完成）
- **And** 每个单元应显示对应熟练度信息
- **And** 当前单元应与当前学习位置一致

#### Scenario: 无历史指标时展示默认值
- **Given** 某课程或单元尚无练习历史
- **When** 用户查看选择面板
- **Then** 系统应显示默认指标文案（如 0 次、0% 或未开始）
- **And** 用户仍可正常选择课程与单元进入学习
